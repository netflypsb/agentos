Setting up a **Pay-per-event (PPE)** trigger on Apify is the most efficient way to monetize an MCP server in 2026. It allows you to charge users for specific tool calls or data results rather than a flat monthly fee.

Here is the step-by-step guide to building, hosting, and monetizing your MCP server on Apify.

---

## 1. Create your Actor from a Template
Apify hosts MCP servers as **Actors**. To start, use the official TypeScript or Python template designed for MCP.

**Using the Apify CLI:**
```bash
apify create my-mcp-server -t ts-mcp-proxy
```
This template includes the Model Context Protocol SDK and the Apify SDK pre-configured for **Streamable HTTP** (which replaced the older SSE transport in April 2026).

---

## 2. Configure PPE in the Apify Console
Before adding code, you must define your "events" and their prices in the dashboard:
1. Go to your **Actor's page** in the Apify Console.
2. Navigate to the **Publication** tab -> **Monetization**.
3. Select **Pay-per-event** as your pricing model.
4. **Define your Events:** Create events like `tool_call_basic` ($0.01) or `advanced_search` ($0.05).
5. **Set the "Synthetic Start" event:** Apify automatically charges a small fee (usually $0.00005) when the Actor starts up to cover compute costs.

---

## 3. Implement the Charging Logic
In your MCP server code, you will use the `Actor.charge()` method. The best practice is to trigger the charge **inside** your tool handlers.

### TypeScript Example
```typescript
import { Actor } from 'apify';
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";

await Actor.init();

const server = new McpServer({
    name: "My Monetized Server",
    version: "1.0.0"
});

server.tool("get_market_data", async (args) => {
    // 1. Attempt to charge the user for the event defined in Console
    const chargeResult = await Actor.charge({ eventName: 'tool_call_basic' });

    // 2. Check if the user has hit their max spending limit
    if (chargeResult.eventChargeLimitReached) {
        return {
            content: [{ type: "text", text: "Error: Run budget exceeded. Please increase your max charge limit." }],
            isError: true
        };
    }

    // 3. Perform the actual tool logic
    const data = await fetchMarketData(args.symbol);
    
    return {
        content: [{ type: "text", text: JSON.stringify(data) }]
    };
});
```

### Python Example
```python
from apify import Actor
from mcp.server.fastmcp import FastMCP

app = FastMCP("My Monetized Server")

@app.tool()
async def premium_search(query: str):
    async with Actor:
        # Trigger the charge
        res = await Actor.charge(event_name='advanced_search')
        
        if res.event_charge_limit_reached:
            return "Monthly budget reached for this tool."

        # Search logic here...
        return f"Results for {query}"
```

## 4. Deploy and Enable "Standby Mode"
MCP servers require a persistent connection. In 2026, Apify uses **Standby Mode** to keep your server "warm" and available via a public URL.

1. **Push your code:** `apify push`.
2. **Enable Standby:** In the **Settings** tab of your Actor, toggle **Standby Mode** to ON.
3. **Get your URL:** Apify will provide a unique endpoint (e.g., `https://my-mcp-server.apify.actor`).
4. **Use Streamable HTTP:** Ensure your MCP client (Claude Desktop, etc.) is configured to use the HTTP transport rather than `stdio`.

---

## Key Tips for 2026
* **Idempotency:** Always pass an `idempotencyKey` in your `Actor.charge()` calls if you are retrying requests to avoid double-charging your users.
* **Budget Transparency:** If a user’s limit is reached, the SDK doesn't automatically kill the process—it just returns `eventChargeLimitReached: true`. You must handle this gracefully in your code to avoid providing free services.
* **Global Taxes:** Apify acts as the "Merchant of Record," meaning they handle all the VAT/GST and local tax filings for the 150+ countries they support.
